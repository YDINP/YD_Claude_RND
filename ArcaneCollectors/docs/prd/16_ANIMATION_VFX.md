## 1. 현재 상태

### 1.1 기존 애니메이션 인프라
| 파일 | 줄 수 | 역할 |
|------|-------|------|
| `src/utils/animations.js` | ~500 | 기본 tween 래퍼 유틸 |
| `src/systems/ParticleManager.js` | 558 | 파티클 이펙트 관리 |
| `src/config/particleConfig.js` | ~300 | 파티클 설정 데이터 |

### 1.2 현재 연출 수준
- **씬 전환**: 즉시 전환 (트랜지션 없음)
- **전투**: 데미지 텍스트 팝업 + 기본 파티클만 존재
- **가챠**: 단순 카드 표시 (등급별 차이 없음)
- **UI**: 버튼 터치 피드백 없음, 모달 즉시 표시/숨김
- **캐릭터**: 정적 원형 + 이니셜 텍스트 (모션 없음)

### 1.3 사용 가능한 Phaser API
- `this.cameras.main.fadeOut()` / `fadeIn()` / `flash()` / `shake()`
- `this.tweens.add()` — 속성 애니메이션
- `this.add.particles()` — 파티클 시스템
- `this.cameras.main.pan()` / `zoom()` — 카메라 이동

## 3. 전투 이펙트 시스템

### VFX-2.1: 스킬 발동 연출 체계

#### 일반 스킬
```
1. [0~200ms]    캐릭터 전진 (x += 50px, ease-out)
2. [200~300ms]  공격 포즈 (scaleX: 1.1)
3. [300~500ms]  히트 이펙트 (분위기별 파티클)
4. [300~600ms]  데미지 텍스트 팝업 (y -40px, 바운스)
5. [500~800ms]  캐릭터 복귀 (ease-in-out)
```

#### 궁극기 (Ultimate Skill)
```
1. [0~300ms]    화면 40% 어두워짐 (오버레이)
2. [300~600ms]  캐릭터 컷인 (화면 중앙 확대, scale 2.0)
3. [600~800ms]  스킬명 텍스트 슬라이드 인
4. [800~900ms]  플래시 이펙트
5. [900~1400ms] 전체화면 이펙트 (분위기별)
6. [1400~1700ms] 데미지 텍스트 (큰 폰트, 바운스)
7. [1700~2000ms] 오버레이 해제, 캐릭터 복귀
```

## 5. UI 마이크로 인터랙션

### VFX-4.1: 버튼 터치 피드백
```js
// 모든 Button 컴포넌트에 적용
onPointerDown: { scaleX: 0.95, scaleY: 0.95, duration: 80, tint: 0xdddddd }
onPointerUp:   { scaleX: 1.0, scaleY: 1.0, duration: 80, clearTint: true }
disabled:      { alpha: 0.5, tint: 0x888888 }
```

### VFX-4.2: 탭/카테고리 전환
- 선택 탭 하단 밑줄 슬라이드 (x 이동, 200ms ease-out)
- 콘텐츠 영역: 새 탭 방향으로 슬라이드 + 페이드 (150ms)
- 비선택 탭 텍스트 색상 변환 (100ms)

### VFX-4.3: 숫자 카운팅 (골드/젬/EXP)
```
변동 감지 시:
1. 이전값→새값 숫자 롤링 (500ms, easeOut)
2. 증가: 녹색 "+" 텍스트 + 빛남 (300ms)
3. 감소: 빨간 "-" 텍스트 + 흔들림 (300ms)
4. 큰 변동(>1000): 속도 가속 + 사운드 pitch up
```

### VFX-4.4: 레벨업 연출
```
1. [0~200ms]    레벨 숫자 scale 1.0→1.5x + 금색 변환
2. [200~500ms]  금색 빛 파티클 폭발 (20개)
3. [300~500ms]  "LEVEL UP!" 텍스트 바운스 등장
4. [500~900ms]  스탯 증가 숫자 순차 표시 (HP↑3, ATK↑2, ... 100ms 간격)
5. [900~1100ms] 새 레벨 숫자 안착 (scale 1.5→1.0x, ease-out)
```

### VFX-4.5: 진화 연출
```
1. [0~500ms]    캐릭터 카드 빛남 (밝기 증가)
2. [500~800ms]  백색 플래시
3. [800~1300ms] 새 등급 프레임으로 변환 + 별 추가
4. [1300~1600ms] 파티클 폭발 (등급 색상)
5. [1600ms~]     새 스탯 표시
```

### VFX-4.6: 토스트 알림 애니메이션
- 등장: 하단에서 상단으로 y -50px 슬라이드 + alpha 0→1 (200ms)
- 유지: 2초
- 퇴장: y -20px 추가 슬라이드 + alpha 1→0 (200ms)
- 다중 토스트: 위로 스택 (기존 토스트 y -= 60px)

### VFX-4.7: 모달 등장/퇴장
- **등장**: 배경 overlay alpha 0→0.6 (200ms) + 모달 scale 0.85→1.0 + alpha 0→1 (200ms)
- **퇴장**: 모달 scale 1.0→0.9 + alpha 1→0 (150ms) + overlay alpha→0 (200ms)
- **배경 터치**: 모달 외부 터치 시 닫기 (설정 가능)

### VFX-4.8: 스크롤 리스트 관성
- HeroListScene, InventoryScene 등 스크롤 영역:
  - 드래그 속도 비례 관성 스크롤 (velocity tracking)
  - 끝단 바운스 (오버스크롤 탄성, 50px max → 복귀)
  - 스크롤바 표시 (드래그 중만, 페이드아웃 300ms)
  - Phaser: `this.input.on('pointermove')` + velocity 계산

### VFX-4.9: 장비 장착/해제 이펙트
- **장착**: 아이콘 슬롯으로 슬라이드 + 빛 섬광
- **해제**: 슬롯에서 인벤토리로 슬라이드
- **비교**: 스탯 차이 표시 (↑녹색 / ↓빨간)

**검증 기준**: 모든 인터랙션 반응 100ms 이내, 60fps 유지

## 6. 캐릭터 전투 모션

### VFX-5.1: 대기(Idle) 모션
- 미세 호흡: scaleY 1.0 ↔ 1.02 (2초 주기, yoyo: true)
- 그림자 동기화: scaleX 1.0 ↔ 0.98
- 전투 대기 중 지속 재생

### VFX-5.2: 공격 모션
```
1. [0~150ms]    전진 (x += 50px, ease-out)
2. [150~250ms]  공격 포즈 (scaleX: 1.1, scaleY: 0.95)
3. [250~350ms]  히트 순간 (scaleX: 1.0, scaleY: 1.0) + 히트 이펙트
4. [350~600ms]  복귀 (x -= 50px, ease-in-out)
5. [600ms~]     idle 모션 재개
```

### VFX-5.3: 피격 모션
```
1. [0~200ms]    x ±5px 흔들림 (3회)
2. [0~300ms]    alpha 1.0→0.5→1.0 (3회 깜빡임)
3. [0~100ms]    빨간 tint (0xFF4444) → 복원
4. [300ms~]     idle 모션 재개
```

### VFX-5.4: 사망 모션
```
1. [0~300ms]    큰 흔들림 (x ±10px, 5회)
2. [300~500ms]  회색 변환 (tint: 0x666666)
3. [500~800ms]  기울기 (angle: 15°, ease-out)
4. [800~1300ms] alpha 1.0→0 (페이드아웃)
5. [1300ms]     오브젝트 비활성화 (visible: false)
```

### VFX-5.5: 버프/힐 수신 모션
- **힐**: y -= 8px (100ms) → 복귀 (200ms) + 녹색 빛남 (200ms)
- **버프**: y -= 5px (100ms) → 복귀 (200ms) + 파란 빛남 (200ms)
- **디버프**: y += 3px (100ms) → 복귀 (200ms) + 보라 빛남 (200ms)

### VFX-5.6: 궁극기 컷인 모션
```
1. [0~200ms]    캐릭터 원본 위치에서 화면 중앙으로 이동 + 확대(scale 2.0)
2. [200~400ms]  포즈 유지 + 배경 블러/어두움
3. [400~600ms]  스킬명 텍스트 좌→우 슬라이드
4. [600~800ms]  캐릭터 투명화 + 이펙트 시작
5. [800ms~]     원래 위치 복귀 (또는 이펙트 중 숨김)
```

**검증 기준**:
- 8체(아군4+적4) 동시 idle 모션 시 60fps 유지
- 사망 모션 완료 후 메모리 해제 (visible:false + inactive)

## 8. 성능 관리

### 8.1 파티클 예산

| 영역 | 동시 파티클 상한 | 수명 | 풀 크기 |
|------|----------------|------|---------|
| 전투 스킬 이펙트 | 50개/스킬 | 0.5~1초 | 100 |
| 가챠 연출 | 100개 | 1~2초 | 150 |
| UI 이펙트 | 20개 | 0.3~0.5초 | 30 |
| 앰비언트 배경 | 30개 | 연속 | 30 |
| **총 동시 최대** | **200개** | - | **310** |

### 8.2 오브젝트 풀링 전략
- `ParticlePool`: 파티클 이미터 재사용 (생성/파괴 대신 reset)
- `DamageTextPool`: 데미지 텍스트 오브젝트 20개 풀 (setText + setPosition)
- `TweenPool`: 동시 tween 최대 15개 제한

### 8.3 프레임 예산
- **목표**: 60fps (16.67ms/frame)
- **애니메이션 예산**: 프레임당 4ms 이하
- **파티클 업데이트**: 프레임당 2ms 이하
- **총 게임 로직**: 프레임당 10ms 이하 (렌더링 6ms 예약)

### 8.4 메모리 관리
- 씬 shutdown() 시 모든 tween 중지 + 파티클 이미터 파괴
- 이벤트 리스너 해제 확인
- 텍스처 캐싱 (같은 파티클 텍스처 공유)
