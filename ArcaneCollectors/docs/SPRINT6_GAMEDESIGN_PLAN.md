## 총평

기술 완성도 70%, 콘텐츠·경험 완성도 20%.
**"시스템의 껍데기는 있으나 내용물이 없는 구조"**
플레이어가 "왜 이 캐릭터를 모으고 싶은가", "왜 다음 스테이지를 가고 싶은가"에 답하지 못하는 상태.

## 섹션 1. 현재 부족한 부분 (우선순위별)

### P0 — 치명적 (게임이 성립하지 않는 수준)

#### 1-1. 캐릭터 콘텐츠 전무
| 항목 | 현황 | 문제 |
|------|------|------|
| 캐릭터 수 | 4개 플레이스홀더 | PRD 목표 91개 대비 4% |
| 캐릭터 아트 | 프로시저럴 실루엣 | 수집 욕구 유발 불가 |
| 캐릭터 스토리 | 없음 | 애착 형성 불가 |

수집 RPG에서 수집하고 싶은 대상이 없는 상태. 가챠 연출/육성 시스템의 의미가 없음.
- 복잡도: H | 임팩트: 고

#### 1-2. 스킬 시스템 미연동
스킬 데이터는 정의되어 있으나 실제 전투에서 발동되지 않음.
모든 캐릭터가 동일한 방식으로 싸워 전략적 차별성이 없음.
- 복잡도: M | 임팩트: 고

---

### P1 — 심각 (장기 리텐션 직결)

#### 1-3. 내러티브/스토리 부재
교단 세계관이 문서로 정리되어 있으나 인게임에서 플레이어가 경험할 수단이 없음.
스테이지 클리어 시 세계관이 전개되지 않아 수집의 맥락이 없음.
- 복잡도: M | 임팩트: 고

#### 1-4. 사운드 시스템 전무
배경음, 전투 효과음, 가챠 연출음 없음.
SSR 등장 팡파레 없어 가챠 쾌감이 절반으로 줄어듦.
로열티 프리 에셋(OpenGameArt, Freesound)으로 빠르게 보완 가능.
- 복잡도: L | 임팩트: 고

---

### P2 — 중요 (플레이 경험 품질)

#### 1-5. 메타 게임 루프 미완성
PvP/길드/이벤트가 시스템으로 존재하나 하나의 주간 루틴으로 묶이지 않음.
"오늘 무엇을 해야 하는지"가 불명확함.
- 복잡도: M | 임팩트: 고

#### 1-6. 시즌/배틀패스 부재
에너지 보석 구매 외 지속 가능한 수익 모델 없음.
- 복잡도: M | 임팩트: 중 (수익 직결)

## 섹션 3. 장기 로드맵 관점 누락 요소

### 3-1. 소셜/커뮤니티 동기
| 누락 요소 | 설명 | 우선순위 |
|-----------|------|---------|
| 파티 구성 공유 카드 | 내 파티를 이미지로 저장/공유 | P2 |
| 가챠 결과 스크린샷 모드 | SSR 뽑기 결과를 SNS 최적화 이미지로 | P2 |
| 길드 공개 랭킹 페이지 | 웹에서 접근 가능한 공개 리더보드 | P3 |

### 3-2. 콘텐츠 고갈 대응
| 장기 콘텐츠 트랙 | 설명 |
|----------------|------|
| 무한 타워 (Abyss Tower) | 기존 타워 시스템을 매주 초기화+심화 |
| 교단 레이드 | 교단별 거대 보스 다인 전투 (길드 연동) |
| 시즌 스토리 | 분기별 신규 스토리 챕터 (3-6챕터 단위) |
| 캐릭터 에피소드 | 개별 캐릭터 친밀도 스토리 (블루아카 방식) |

### 3-3. 경제 밸런싱 문서화 (미작성)
아래 세 가지 최소 문서화 필요:
- 가챠 pity 달성까지의 기대 과금액
- 자연 에너지 회복으로 주간 가챠 횟수
- 무과금/소과금/중과금 진행 속도 비교표

### 3-4. PWA 홈 화면 추가 전면 활성화
이미 PWA 지원 완료 상태 → "홈 화면에 추가" 배너를 게임 내 전면에 노출하여 재방문율 향상.

## 섹션 4. 즉시 개선 효과 큰 UX 항목

### 4-1. 메인 메뉴 씬
| 항목 | 복잡도 | 개선 방향 |
|------|-------|---------|
| 로딩 화면 브랜딩 | L | 게임 타이틀 + 교단 엠블럼 5종 로테이션 |
| 메인 메뉴 배경 애니메이션 | M | 정적 배경 → 파티클/빛 효과 |
| 버튼 10개 배치 재구성 | M | 핵심 3개 강조, 나머지 서브 메뉴로 이동 |
| 신규 플레이어 온보딩 화살표 | L | 첫 접속 시 "이것부터 해보세요" 안내 |

### 4-2. 가챠 연출 강화
| 체크 항목 | 개선 방향 |
|-----------|---------|
| SSR 등장 전 긴장감 | 2-3초 서스펜스 빌드업 (검은 화면, 느린 등장) |
| 등급별 연출 차별화 | N 평범 / R 빛 효과 / SR 폭발 / SSR 전체 화면 |
| 10연차 결과 일람 화면 | 10개 카드 동시 표시 + 하이라이트 |
| 도감 달성률 연동 피드백 | "컬렉션 도감 XX% 달성!" 표시 |

### 4-3. 전투 화면
| 항목 | 복잡도 | 개선 방향 |
|------|-------|---------|
| **전투 배속 버튼** | L | 1x / 2x / 4x 배속 (방치형 필수 기능 누락) |
| 데미지 숫자 폰트 | L | Orbitron으로 대형 표시 |
| 스킬 발동 알림 배너 | L | 화면 상단 슬라이드인 "○○ 스킬 발동!" |
| 자동/수동 전환 명확화 | L | 현재 모드 상태 항상 표시 |

### 4-4. 캐릭터 도감
| 체크 항목 | 개선 방향 |
|-----------|---------|
| 미수집 캐릭터 실루엣 | "저 검은 실루엣이 누굴까" 호기심 유발 |
| 교단별 수집률 표시 | "올림포스 교단 3/10 수집" 완성 욕구 |
| 즐겨찾기/정렬 기능 | 91캐릭터 대비 필터 강화 |

## 스프린트 실행 순서 로드맵

```
즉시 (1-2주)   Sprint B: 사운드 통합
               + 스킬 배틀 연동 (Sprint A 일부)
               → 비용 최소, 체감 개선 최대

단기 (3-4주)   Sprint A: 교단별 대표 캐릭터 5명
               Sprint C: 일일 퀘스트 + 출석 체크

중기 (5-8주)   Sprint D: 프롤로그 스토리 + 비주얼 노벨 씬 시스템
               + 메인 메뉴 UX 개선

장기 (2개월+)  캐릭터 콘텐츠 확장 (교단별 10명씩 순차)
               + 시즌 시스템 + 배틀패스
```
