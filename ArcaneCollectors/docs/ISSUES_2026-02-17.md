# ArcaneCollectors 이슈 목록 (2026-02-17)

브라우저 테스팅 + 코드 분석 에이전트 4개 통합 결과

## P0 — 크래시/기능불가 (5건)

### 1. getStageCost() 크래시
- **파일**: `src/scenes/StageSelectScene.js:641`
- **문제**: `getStageCost("1-1")` 호출 시 EnergySystem이 "NORMAL"/"ELITE"/"BOSS" 타입만 허용 → throw Error
- **수정**: stageId → stageType 변환 로직 추가

### 2. sweepSystem.recordStageClear() 미호출
- **파일**: `src/scenes/BattleScene.js`
- **문제**: 전투 승리 후 SweepSystem에 클리어 기록을 남기지 않음 → 소탕 시스템 전체 불가
- **수정**: 전투 승리 핸들러에 `sweepSystem.recordStageClear(stageId, stars)` 추가

### 3. 가챠 천장 카운터 불일치
- **파일**: `src/systems/GachaSystem.js:265`, `src/utils/SaveManager.js:607`
- **문제**: 10연차 중간 SSR 획득 시, SSR 이후 남은 회차가 천장에 누적되지 않음
- **수정**: `updateGachaCounter(count, gotSSR)` → `updateGachaCounter(finalPityCount)` 로 변경

### 4. getBaseStars() 모든 캐릭터 1성 고정
- **파일**: `src/utils/SaveManager.js:391`
- **문제**: 캐릭터 ID `hero_001` → `_` 앞 `'hero'`를 등급으로 파싱 → starsMap에 없어 항상 1 반환
- **수정**: characters.json에서 rarity 필드 직접 참조하도록 변경

### 5. 설정 팝업 z-index 투과
- **파일**: `src/components/popups/SettingsPopup.js`
- **문제**: 설정 팝업 뒤로 메인 메뉴 "내 파티" 영역이 보임 (overlay가 상단 미커버)
- **수정**: PopupBase overlay depth 또는 SettingsPopup 레이아웃 수정

## P1 — 기능 누락 (3건)

### 6. 이벤트 던전 코드 변수명 노출
- **파일**: `src/components/popups/EventDungeonPopup.js`
- **문제**: `dragon_scale`, `shadow_fragment` 등 코드 변수명이 UI에 그대로 표시
- **수정**: 한글 표시명 매핑 추가

### 7. 가챠 UI 하드코딩
- **파일**: `src/scenes/GachaScene.js`, `src/components/popups/GachaPopup.js`
- **문제**: 소환 비용(300, 2700), 천장(90) 등이 GachaSystem 상수 대신 하드코딩
- **수정**: `GachaSystem.SINGLE_COST`, `GachaSystem.PITY_THRESHOLD` 등 참조

### 8. TowerSystem EXP 미지급
- **파일**: `src/systems/TowerSystem.js:186`
- **문제**: 탑 클리어 시 골드는 지급되지만 EXP는 TODO 주석만 있음
- **수정**: `ProgressionSystem.addExp()` 연동

- **문제**: getMoodBonus() 내 상성 매트릭스가 MoodSystem과 중복 정의
- **수정**: MoodSystem 싱글톤 호출로 통합

### 12. personalities.json 데드파일

- **문제**: 구형 5종 성격 시스템 파일이 import 없이 잔존
