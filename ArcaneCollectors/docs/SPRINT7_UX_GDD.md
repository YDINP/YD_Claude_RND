## 섹션 1. 메인 메뉴 UX 재구성

### 1-1. 현재 문제점 (Sprint 6 분석)

- 버튼 10개가 동일 위계로 나열 → 신규 유저가 무엇부터 해야 할지 모름
- 핵심 콘텐츠(스테이지/가챠/파티)와 설정/기타가 동일 공간 점유
- 세계관 분위기를 메인 메뉴에서 전혀 느낄 수 없음

### 1-2. 재구성 원칙

**허브 구조 + 3강조 버튼 + 서브 메뉴 콜랩스:**

```
[메인 메뉴 화면]

  ┌─────────────────────────────────┐
  │         [배경 애니메이션]           │
  │   (교단 엠블럼 파티클 + 별자리)        │
  │                                 │
  │       ArcaneCollectors          │
  │       균열의 수집가                │
  │                                 │
  │  ┌──────────────────────────┐   │
  │  │ [P1] 전투 / 스테이지 진행  │   │  ← 최강조 (가장 크고 밝음)
  │  └──────────────────────────┘   │
  │                                 │
  │  ┌────────────┐ ┌────────────┐  │
  │  │ [P2] 가챠  │ │[P2] 파티   │  │  ← 보조 강조 (중간 크기)
  │  └────────────┘ └────────────┘  │
  │                                 │
  │  ── ── ── ── ── ── ── ── ──     │
  │  일일 퀘스트 | 도감 | PvP | 길드   │  ← 서브 아이콘 (소형)
  │  타워 | 이벤트 | 상점 | 설정      │
  └─────────────────────────────────┘

  [하단 HUD]
  에너지: 85/120 | 보석: 1,240 | 새 알림: 3
```

### 1-3. 메인 메뉴 버튼 계층 구조

| 계층 | 버튼 | 크기 | 위치 | 강조 방식 |
|------|------|------|------|---------|
| P1 (최강조) | 전투/스테이지 | 전체 너비 70%, 높이 60px | 중앙 상단 | 빛나는 테두리 + 펄스 애니메이션 |
| P2 (보조) | 가챠, 파티 편성 | 각 45%, 높이 50px | 중앙 하단 1열 | 교단 컬러 아이콘 |
| P3 (서브) | 일일 퀘스트, 도감, PvP, 길드, 타워, 이벤트, 상점, 설정 | 아이콘 크기 36px | 하단 2행 | 작은 텍스트 + 아이콘 |

### 1-4. 신규 플레이어 온보딩 안내 (첫 접속 시)

**단계별 화살표 가이드 (3회 접속 이내만 표시):**

| 접속 순서 | 강조 요소 | 안내 텍스트 |
|---------|---------|----------|
| 1회차 접속 | [전투/스테이지] 버튼 | "수집가님, 첫 미정령이 기다리고 있어요!" |
| 1회차 접속 (전투 후) | [가챠] 버튼 | "전투 보상으로 소환을 해볼까요?" |
| 2회차 접속 | [일일 퀘스트] 아이콘 | "매일 퀘스트를 완료하면 소환 재화를 얻어요!" |

**조건**: 튜토리얼 완료 + 챕터 1-1 클리어 후 메인 메뉴 복귀 시 표시

### 1-5. 일일 요약 위젯

메인 메뉴 하단 or 상단 고정 영역:

```
┌────────────────────────────────────┐
│ 오늘의 퀘스트   ■■■□□  3/5 완료      │
│ 출석 체크       ✓ 완료 (6일 연속)     │
│ 에너지         85/120 (2시간 후 완충) │
│ 이벤트         [교단 시련] 곤륜 오늘!  │
└────────────────────────────────────┘
```

## 섹션 2. 가챠 연출 강화 UX

### 2-1. 현재 문제점

- 등급별 연출 차별화 없음 → SSR이 나와도 감흥 없음
- 10연차 후 결과 요약 화면 없음 → 어떤 등급이 나왔는지 한눈에 파악 불가
- SSR 서스펜스 빌드업 없음 → 기대감 없이 결과 표시

### 2-2. 등급별 연출 설계

**단일 소환 연출 흐름:**

```
[소환 버튼 터치]
      ↓
[0.5초] 화면 페이드 다크
      ↓
등급별 분기 ────────────────────────────────────────────────────
│ N (일반)                │ R (레어)                │ SR (슈퍼레어)           │ SSR (최고)
│ 0.3초 빠른 등장          │ 0.8초 별빛 이펙트          │ 1.5초 교단 컬러 폭발     │ 3.0초 서스펜스 빌드업
│ 일반 등장음              │ 빛 효과 + 이펙트음          │ 화면 진동 + 임팩트음     │ 검은 화면 + 느린 출현
│ 캐릭터 실루엣 표시         │ 교단 컬러 실루엣            │ 전체화면 교단 이펙트      │ 전체화면 교단 컬러 풀발산
│                         │                           │                        │ SSR 팡파레 사운드
└─────────────────────────────────────────────────────────────────────────────

[캐릭터 카드 등장]
      ↓
[신규 캐릭터 시] "NEW!" 배지 + 교단 소개 텍스트 팝업
[중복 캐릭터 시] 각성 소재 자동 변환 안내
      ↓
[계속 소환 or 결과 확인 선택]
```

### 2-3. SSR 서스펜스 연출 상세

**SSR 등장 시 3단계 빌드업:**

| 단계 | 시간 | 연출 | 사운드 |
|------|------|------|------|
| 빌드업 | 0~1.5초 | 화면 서서히 어두워짐, 별자리 파티클 등장 | 저음 드론 사운드 점점 커짐 |
| 클라이맥스 준비 | 1.5~2.0초 | 교단별 컬러 균열이 화면에 퍼짐 | 균열음 + 긴장감 사운드 |
| 등장 | 2.0~3.0초 | 교단 컬러 전체 폭발 → 캐릭터 실루엣 출현 → 이름 표시 | SSR 팡파레 (교단별 다른 음악) |

**스킵 기능**: 화면 터치 시 남은 연출 스킵 → 결과 바로 표시 (연속 가챠 효율)

### 2-4. 10연차 결과 일람 화면

**10연차 완료 후 결과 화면:**

```
┌─────────────────────────────────────────────────────┐
│                   소환 결과                             │
│                                                       │
│  [카드1] [카드2] [카드3] [카드4] [카드5]                │
│   SR    N     R     N     SR                          │
│                                                       │
│  [카드6] [카드7] [카드8] [카드9] [카드10]               │
│   R     N     N     SSR★  R                          │
│                                                       │
│  ┌───────────────────────────────────────────┐       │
│  │ ★ SSR 획득! 아테나 (Olympus)              │       │  ← 별도 강조 영역
│  │ 컬렉션 도감 달성률: 12% → 13%             │       │
│  └───────────────────────────────────────────┘       │
│                                                       │
│      [다시 소환 (재화 있을 시)]  [확인]                   │
└─────────────────────────────────────────────────────┘
```

**하이라이트 규칙:**
- SSR 카드: 금색 테두리 + 빛나는 이펙트
- SR 카드: 교단 컬러 테두리
- 신규 캐릭터: "NEW!" 배지
- 중복 캐릭터: "→ 각성 소재 ×1" 표시

┌─────────────────────────────────────────────────────┐
│  일일 퀘스트                    초기화: 04:59:23        │
│                                                       │

- 완주 보상은 5개 모두 완료 시 별도 강조 애니메이션 표시

## 섹션 4. 소셜 공유 기능 UX

### 4-1. 파티 카드 이미지 저장

**기능 위치**: 파티 편성 화면 → 우측 상단 [공유] 버튼

**파티 카드 이미지 구성:**

```
┌──────────────────────────────────────────┐
│                                          │
│    ⬡ ArcaneCollectors ⬡                 │
│    "균열의 수집가"                          │
│                                          │
│  [캐릭1]  [캐릭2]  [캐릭3]  [캐릭4]        │
│  아테나    시그룬   모리간   흑제            │
│  Olympus Valhalla Avalon  Kunlun        │
│  SSR★    SR      SSR★    SR            │
│                                          │
│  파티 전투력: 128,400                      │
│  교단 시너지: Olympus+Avalon (2+2)         │
│                                          │
│  arcane-collectors.vercel.app           │
└──────────────────────────────────────────┘
```

**저장 플로우:**
1. [공유] 버튼 탭 → 이미지 생성 (0.5초 내)
2. 저장 방식 선택: [갤러리 저장] / [링크 복사] / [SNS 바로 공유]
3. SNS 바로 공유: 트위터/X, 인스타그램 스토리 포맷 자동 조정

### 4-2. SNS 최적화 가챠 결과 공유

**SSR 획득 시 공유 유도 흐름:**

```
[SSR 획득]
      ↓
결과 화면에 [SNS 공유] 버튼 표시 (선택적)
      ↓
공유 이미지 자동 생성:

┌──────────────────────────────┐
│                              │
│     ✨ SSR 획득! ✨           │
│                              │
│     [아테나 실루엣]            │
│     아테나 (Olympus)          │
│     "지혜의 여신"              │
│                              │
│  ArcaneCollectors             │
│  같이 수집해볼까요?             │
│  → 링크                       │
└──────────────────────────────┘
```

**UX 원칙:**
- 공유는 항상 선택적 (강제 공유 없음, 다크패턴 방지)
- 공유 시 링크 클릭자에게 "첫 소환 무료 재화 ×3" 보상 (레퍼럴 효과)
- 공유 이미지 생성은 HTML Canvas API 활용 (외부 API 없음)

## 섹션 6. 다크패턴 방지 체크리스트

```
□ 가챠 결과 확인 없이 자동 연속 소환되는 버튼이 없는가?
□ 에너지 소진 시 즉시 결제 팝업이 전면에 등장하지 않는가?
□ 무료 소환권과 유료 소환권의 버튼이 동일 크기/색으로 혼동되지 않는가?
□ SSR 서스펜스 연출이 항상 스킵 가능한가?
□ 출석 체크 알림이 동의 없이 자동 발송되지 않는가?
□ 소셜 공유가 강제 또는 보상 강요 없이 항상 선택적인가?
□ 가챠 확률 정보가 소환 버튼 터치 전 접근 가능한 위치에 있는가?
□ 구매 확인 팝업 없이 즉시 결제되는 버튼이 없는가?
```

## 섹션 7. 접근성 가이드라인

| 항목 | 기준 |
|------|------|
| 최소 버튼 크기 | 44×44px (모바일 터치 기준) |
| 텍스트 최소 크기 | 12pt (게임 UI 기준) |
