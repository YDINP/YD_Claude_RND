# 번짐(Bleed) — 미니 GDD

---

## 1. 한 줄 정의

인접 셀로 색이 번져나가는 확산 규칙을 머릿속으로 시뮬레이션하여, 최소 주입 횟수로 판 전체를 목표 상태로 물들이는 캐주얼 퍼즐 게임.

---

## 2. 핵심 루프

```
[관찰] 판 상태 + 목표 패턴 확인
  → [계획] 어디에 무슨 색을 주입할지 머릿속 시뮬레이션
    → [주입] 셀 탭 → 색 선택 → 확산 애니메이션 재생 (0.8~1.5초)
      → [판정] 번짐 결과 확인 → 목표 달성 여부 체크
        → [반복] 남은 주입 횟수 내에서 반복
          → [완료] 별점 평가 (주입 횟수 기준 1~3성)
```

| 항목 | 수치 |
|------|------|
| 세션당 플레이 시간 | 2~5분 (스테이지 1개) |
| 스테이지당 주입 횟수 | 3~12회 (난이도별) |
| 번짐 애니메이션 1회 | 0.8~1.5초 (연쇄 시 누적) |
| 일일 목표 플레이 | 15~20분 (5~8스테이지) |

**세부 플로우:**
1. 스테이지 시작: NxN 격자에 일부 셀이 이미 색칠된 초기 상태 표시. 우측에 목표 패턴 썸네일.
2. 플레이어가 빈 셀 또는 기존 색 셀을 탭 → 하단 팔레트에서 색 선택 → 주입.
3. 주입된 색이 번짐 규칙에 따라 인접 셀(4방향)로 확산. 확산은 동시가 아니라 파동처럼 1턴에 1칸씩 진행.
4. 번짐 도중 다른 색과 만나면 색 화학 규칙 적용(섞임/상쇄/생성).
5. 번짐이 멈추면(더 이상 확산 불가) 1주입 소모. 남은 횟수 차감.
6. 판 상태가 목표 패턴과 일치하면 클리어. 남은 주입 횟수에 따라 1~3성.

---

## 3. 색다름 포인트

### 차이 1: 매칭이 아니라 확산 시뮬레이션

매치3는 "같은 것 3개 맞추기"라는 패턴 인식 게임이다. 번짐은 "한 점에서 주입한 색이 규칙에 따라 어디까지 어떻게 퍼지는가"를 예측하는 시뮬레이션 게임이다. 플레이어의 사고 방식이 근본적으로 다르다. 매치3가 "찾기"라면, 번짐은 "예측하기"다.

### 차이 2: 입력이 극소, 결과가 극대

매치3는 매번 스왑/탭으로 소규모 변화를 반복한다. 번짐은 단 1회 주입으로 판 절반이 바뀔 수 있다. 3~5회의 주입으로 전체 판이 완성되는 "나비효과" 쾌감이 핵심이다. 한 수의 무게가 매치3보다 압도적으로 크다.

### 차이 3: 색 자체가 게임 메커니즘

매치3에서 색은 구분용 시각 기호일 뿐이다. 번짐에서 색은 물리적 실체다. 빨강+파랑=보라, 빨강+노랑=주황처럼 색이 섞이고, 보색(빨강 vs 초록)은 상쇄되어 사라진다. 색 자체의 속성이 퍼즐 난이도와 풀이를 결정한다.

## 4. 난이도 확장 설계

### 4-1. 색 화학 규칙 (Color Chemistry)

**기본 색 3원색:** 빨강(R), 파랑(B), 노랑(Y)

| 조합 | 결과 | 이름 | 규칙 |
|------|------|------|------|
| R + B | 보라(V) | 혼합 | 두 색이 같은 셀에서 만나면 혼합색 생성 |
| R + Y | 주황(O) | 혼합 | 동일 |
| B + Y | 초록(G) | 혼합 | 동일 |
| R + G | 소멸(X) | 보색 상쇄 | 두 색 모두 소멸, 셀이 빈 상태로 돌아감 |
| B + O | 소멸(X) | 보색 상쇄 | 동일 |
| Y + V | 소멸(X) | 보색 상쇄 | 동일 |
| 2차색 + 2차색 | 덮어쓰기 | 우선 규칙 | 나중에 도착한 색이 덮어씀 (번짐 방향이 중요) |
| 같은 색 + 같은 색 | 강화(+) | 농도 증가 | 농도 2단계: 연한(1) → 진한(2). 진한 색은 번짐 범위 +1 |

**번짐 규칙:**
- 기본 번짐: 주입 지점에서 4방향(상하좌우)으로 1턴에 1칸씩 확산
- 번짐 범위: 기본 2칸. 농도 "진한" 상태면 3칸
- 번짐 정지 조건: (a) 범위 한계 도달, (b) 벽/장애물, (c) 보색 상쇄로 소멸
- 번짐 순서: 주입 순서대로 처리. 동시 충돌 시 먼저 주입된 색 우선

### 4-2. 장애물 셀 (Obstacle Cells)

| 장애물 | 효과 | 등장 시기 |
|--------|------|----------|
| 벽(Wall) | 번짐 완전 차단. 색이 통과 불가 | 스테이지 11~ |
| 바위(Rock) | 1회 번짐으로 파괴 가능. 파괴 시 색 소모 | 스테이지 21~ |
| 거울(Mirror) | 번짐 방향을 90도 반사 | 스테이지 31~ |
| 흡수체(Absorber) | 색을 흡수하여 번짐 범위 -1 | 스테이지 41~ |
| 증폭기(Amplifier) | 통과하는 색의 번짐 범위 +2 | 스테이지 51~ |

### 4-3. 목표 유형 (Goal Types)

| 목표 | 설명 | 등장 시기 |
|------|------|----------|
| 단색 채우기 | 판 전체를 지정색 1개로 물들이기 | 스테이지 1~ |
| 패턴 매칭 | 제시된 색 배치와 동일하게 만들기 | 스테이지 6~ |
| 색 제거 | 특정 색을 보색 상쇄로 모두 제거 | 스테이지 16~ |
| 농도 맞추기 | 특정 셀을 "진한" 농도로 만들기 | 스테이지 26~ |
| 혼합 도달 | 2차색(보라/주황/초록)을 정해진 위치에 생성 | 스테이지 36~ |

### 4-4. 난이도별 레벨 예시

#### 초급 (스테이지 1~10)

**예시 A — "첫 번짐" (스테이지 1)**
- 격자: 3x3, 전체 비어있음
- 팔레트: 빨강(R) 1색만 제공
- 목표: 판 전체를 빨강으로 채우기
- 주입 제한: 3회
- 풀이: 중앙(2,2)에 R 주입 → 4방향으로 2칸 번짐 → 1회로 9칸 전부 커버. 3성 클리어.
- 교육: 번짐의 기본 규칙(4방향, 2칸 범위) 학습

**예시 B — "두 색 만남" (스테이지 5)**
- 격자: 4x4, 좌측 2열이 빨강(R)으로 채워져 있음
- 팔레트: 파랑(B)
- 목표: 판 전체를 보라(V)로 만들기
- 주입 제한: 2회
- 풀이: (1,3)에 B 주입 → 좌측으로 번지며 R과 만나 V 생성. (4,3)에 B 주입 → 나머지 커버.
- 교육: R+B=V 혼합 규칙 학습

#### 중급 (스테이지 11~30)

**예시 C — "벽 너머" (스테이지 14)**
- 격자: 5x5, 3행 전체가 벽(Wall)으로 가로막힘
- 팔레트: 빨강(R), 파랑(B)
- 목표: 상단 2행 = 빨강, 하단 2행 = 파랑
- 주입 제한: 4회
- 풀이: 벽이 번짐을 차단하므로 상/하를 독립적으로 처리. 상단에 R 2회, 하단에 B 2회.
- 핵심: 벽을 이용한 영역 분리 개념

**예시 D — "보색 청소" (스테이지 22)**
- 격자: 5x5, 전체가 빨강(R). 중앙(3,3)만 초록(G)
- 팔레트: 초록(G)
- 목표: 판 전체를 빈 상태(소멸)로 만들기
- 주입 제한: 5회
- 풀이: 초록(G)을 빨강(R) 영역에 전략적으로 주입 → R+G 보색 상쇄로 양쪽 소멸. 소멸의 연쇄를 이용.
- 핵심: 보색 상쇄 메커니즘의 전략적 활용

#### 고급 (스테이지 31~)

**예시 E — "거울 반사 퍼즐" (스테이지 35)**
- 격자: 6x6, (3,1)과 (3,6)에 거울 배치 (번짐을 90도 아래로 반사)
- 팔레트: 빨강(R), 노랑(Y)
- 목표: 특정 4셀을 주황(O)으로 만들기, 나머지는 비어있어야 함
- 주입 제한: 3회
- 풀이: 거울 반사를 이용해 R과 Y의 번짐 경로가 정확히 목표 셀에서 교차하도록 계산.
- 핵심: 번짐 경로의 공간 추론 + 거울 메커니즘

**예시 F — "체인 리액션" (스테이지 42)**
- 격자: 7x7, 다수 셀에 서로 다른 색이 산재. 증폭기 2개, 바위 3개 배치
- 팔레트: 3원색 모두
- 목표: 전체를 보라(V)로 통일
- 주입 제한: 4회
- 풀이: 증폭기를 통과시켜 번짐 범위를 극대화하고, 바위를 깨서 경로를 확보하며, 최종적으로 R+B가 정확히 전체에서 만나도록 설계. 1수의 실수가 전체 실패로 이어짐.
- 핵심: 복합 메커니즘 통합, 긴 연쇄 시뮬레이션

## 6. 아트 & 기술 비용

### 개발 난이도 평가

| 항목 | 난이도 | 비고 |
|------|--------|------|
| 핵심 번짐 엔진 | 중 | BFS/파동 시뮬레이션. 격자 기반이라 알고리즘 자체는 단순. 색 화학 규칙 조합 테스트가 핵심 비용 |
| 번짐 애니메이션 | 중~상 | 셀 단위 색 전환은 쉬우나, "물감이 퍼지는" 유체 느낌 쉐이더는 비용 높음. MVP에선 셀 단위 페이드 전환으로 대체 가능 |
| 레벨 에디터/생성 | 중 | 내부 에디터 필수. 자동 난이도 검증(최소 주입 횟수 계산)에 역방향 솔버 필요 |
| UI/HUD | 하 | 격자 + 팔레트 + 카운터. 최소 UI |
| 콘텐츠 볼륨 | 중 | 레벨 자체는 JSON 데이터. 솔버가 있으면 반자동 생성 가능 |

### 번짐 애니메이션 단계별 비용

| 단계 | 구현 | 비용 | 품질 |
|------|------|------|------|
| MVP | 셀 단위 색상 lerp (0.3초 페이드) | 1~2일 | 기능적. 번짐 느낌 약함 |
| v1.0 | 셀 경계에서 그라디언트 블러 + 파동 타이밍 | 3~5일 | 수채화 느낌 시작 |
| v2.0 | Fragment shader 유체 시뮬(Voronoi noise 기반) | 1~2주 | 실제 잉크 번짐 수준 |

### 콘텐츠 생산 파이프라인

```
[레벨 에디터] → 격자/장애물/초기색/목표 배치
  → [솔버] → 최소 주입 횟수 자동 계산 + 풀이 경로 검증
    → [난이도 태깅] → 솔버 결과 기반 자동 분류 (초급/중급/고급)
      → [플레이테스트] → 내부 QA 3명 이상 검증
        → [출시] → JSON 데이터로 서버 배포 (앱 업데이트 불필요)
```

목표 생산 속도: 솔버 완성 후 주당 20~30스테이지 제작 가능.

## 7. 핵심 리스크

### 리스크 1: "결국 매치3이랑 뭐가 달라?" — 차별화 인식 실패

**위험:** 스토어 스크린샷만 보면 "컬러 퍼즐"로 매치3류와 묶여버림. 유저가 설치 전에 차이를 인지 못함.

**완화책:**
- 스토어 영상에서 "1탭으로 판 전체가 변하는" 연쇄 장면을 첫 3초에 배치. 매치3에 없는 시각적 임팩트 강조.
- 게임 이름/부제로 메커니즘 직접 명시: "번짐 — 색이 퍼지는 퍼즐"
- 온보딩 첫 3스테이지에서 "1회 주입 → 판 전체 변화" 체험을 반드시 포함. 매치3과 다르다는 인식을 게임 내에서 즉시 심어줌.

### 리스크 2: 머릿속 시뮬레이션 과부하 — 캐주얼 유저 이탈

**위험:** 번짐 결과를 예측하려면 BFS를 머릿속으로 돌려야 한다. 중급 이상에서 색 3종 + 장애물 조합이 나오면 캐주얼 유저가 "어렵다"고 느끼고 이탈.

**완화책:**
- "미리보기" 기능: 주입 전 탭을 길게 누르면 번짐 예상 범위를 반투명 오버레이로 표시. 시뮬레이션 부담을 시각적으로 보조.
- 난이도 곡선을 매우 완만하게 설계. 스테이지 1~20까지는 2색 + 장애물 0으로 제한. 규칙을 1개씩 점진 도입.
- "자동 풀이 감상" 모드: 막히는 스테이지에서 광고 시청 후 최적 풀이를 애니메이션으로 보여줌. 학습 효과.

### 리스크 3: 레벨 소모 속도 > 생산 속도

**위험:** 핵심 루프가 짧아(2~5분) 콘텐츠 소모가 빠르다. 솔버 기반 반자동 생산이 안 되면 콘텐츠 고갈.

**완화책:**
- MVP 단계에서 솔버(역방향 탐색)를 반드시 구현. 이것이 없으면 레벨 생산이 수작업이 되어 지속 불가.
- "나만의 퍼즐" UGC(유저 생성 콘텐츠)를 v1.1에서 도입. 유저가 레벨을 만들고 공유하면 콘텐츠 자급.
- 데일리/주간 챌린지는 솔버가 자동 생성한 판에서 큐레이션만 하는 방식.

## 9. 한 줄 평결

| 항목 | 점수 | 근거 |
|------|------|------|
| 색다름 | 4/5 | 확산 시뮬레이션 + 색 화학은 모바일 퍼즐 시장에서 거의 미개척. 단, "색 퍼즐"이라는 외형이 기존작과 혼동될 여지 있음 |
| 캐주얼 접근성 | 3.5/5 | "탭 한 번 → 색 번짐"은 직관적이나, 중급 이후 머릿속 시뮬레이션 부담이 캐주얼 한계선을 시험함. 미리보기 기능이 핵심 완화책 |
| 개발 난이도 | 3/5 | 격자 기반 BFS 엔진 자체는 단순하나, 솔버(역방향 탐색) + 유체 느낌 쉐이더가 공수를 끌어올림. MVP는 2주 가능 |
| 바이럴 잠재력 | 4/5 | 번짐 애니메이션의 시각적 만족감이 높아 공유 욕구를 자극. 데일리 랭킹 + 타임랩스 공유가 루프를 형성. UGC가 붙으면 폭발력 있음 |

**종합: 14.5 / 20** — 메커니즘의 참신함과 시각적 바이럴성이 강점. 캐주얼 접근성과 솔버 개발이 성패를 가르는 관건. 미리보기 기능을 핵심 UX로 우선 구현하고, 번짐 애니메이션 품질에 초기 투자를 집중하면 매치3 레드오션에서 차별화된 포지션 확보 가능.
